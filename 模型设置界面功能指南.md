## 三、数据模型 (`AIModelConfig`)

| 字段 | 类型 | 说明 |
|------|------|------|
| `Id` | int | 主键，自增 |
| `Name` | string | 内部名称（如 `deepseek_v3`） |
| `DisplayName` | string | 显示名称（如 `DeepSeek-V3`） |
| `ApiUrl` | string | API地址（如 `https://api.deepseek.com`） |
| `ApiKey` | string | 密钥 |
| `DefaultModel` | string | 模型ID（如 `deepseek-chat`） |
| `Provider` | string | 提供商（OpenAI/Google/Anthropic/DeepSeek/Qwen/Other） |
| `Purpose` | string | 用途：`Summary`(摘要生成) / `DeepResearch`(深度研究) |
| `IsDefault` | bool | 是否为默认模型 |
| `CreatedAt` | DateTime | 创建时间 |
| `UpdatedAt` | DateTime | 更新时间 |

## 四、核心功能

### 1. 模型列表显示
- 左侧列表展示所有已配置的模型
- 显示星号标记默认模型 (★)
- 按用途分组显示标签 `[Summary]` / `[DeepResearch]`

### 2. 新增模型
- 点击"新增模型"按钮
- 右侧编辑面板变为可编辑状态
- 默认值：API地址 `https://api.openai.com/v1`，默认模型 `gpt-4o`

### 3. 编辑模型
- 先选中列表中的模型，点击"修改"按钮
- 或直接双击列表项
- 支持修改所有字段

### 4. 测试连接
- 点击"测试连接"按钮
- 显示加载状态 "正在测试连接..."
- 返回成功/失败结果（绿色✓ / 红色✗）

### 5. 保存/删除
- 保存后自动刷新模型列表
- 删除前无二次确认，直接删除

### 6. 用途分类
| 用途 | 说明 |
|------|------|
| `Summary` | 摘要生成（用于提炼策略摘要） |
| `DeepResearch` | 深度研究（用于深度分析市场） |

## 五、现有实现 vs 优化建议

| 功能 | 现有实现 | 优化建议 |
|------|----------|----------|
| 模型选择 | 两个独立下拉框 | 可合并为一个下拉框，带用途标签 |
| 用途分类 | 通过 `Purpose` 字段区分 | 可添加颜色标识区分 |
| 删除确认 | 无确认 | 添加确认对话框 |
| 导入/导出 | 无 | 可添加 JSON 导入导出功能 |
| 模型排序 | 按添加顺序 | 可按用途/提供商/名称排序 |
