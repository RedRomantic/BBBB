# 飞书消息转发功能说明

## 一、功能概述

本系统提供飞书消息推送功能，支持将客户端消息转发到飞书群或用户。整个流程包括：**客户端 API 请求 → 数据库记录 → 后台扫描 → 飞书推送**。

### 核心特性
- **API 接收**：客户端通过 REST API 提交消息请求
- **后台自动扫描**：定时扫描待处理消息队列
- **双模式推送**：支持文字消息和图片消息
- **多终端配置**：可为不同终端独立配置推送参数
- **状态追踪**：完整的消息发送状态管理

---

## 二、完整工作流程

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   客户端      │     │   API 服务    │     │   数据库      │     │  后台扫描服务  │
│              │────▶│              │────▶│              │◀────│              │
│ (POST 请求)   │     │ (记录消息)    │     │ (消息队列)    │     │ (定时任务)    │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
                                                                     │
                                                                     ▼
                                                              ┌──────────────┐
                                                              │   飞书 API    │
                                                              │              │
                                                              │ (推送消息)    │
                                                              └──────────────┘
```

### 2.1 流程说明

| 步骤 | 组件 | 说明 |
|------|------|------|
| 1 | 客户端 | 调用 POST `/api/feishu/message` 提交消息请求 |
| 2 | API 服务 | 验证请求，创建消息记录到数据库，状态为 `Pending` |
| 3 | 数据库 | 存储消息记录，等待处理 |
| 4 | 后台扫描 | 定时扫描 `Pending` 状态的消息 |
| 5 | 推送服务 | 根据终端配置调用飞书 API 发送消息 |
| 6 | 状态更新 | 发送成功后更新状态为 `Sent`，失败则为 `Failed` |

## 三、客户端 API

### 3.1 创建消息请求

客户端通过此接口提交飞书推送请求，支持**文字消息推送**和**数据获取推送**两种模式。

#### 模式一：直接推送文字

**请求体**

```json
{
    "terminalId": "terminal_001",
    "messageType": "push",
    "method": "text"
}
```

#### 模式二：获取数据后推送（推荐）

通过 `sourceId` 参数指定消息源，系统会自动调用配置的 API 获取数据并格式化后推送。

**请求体**

```json
{
    "terminalId": "t1",
    "messageType": "push",
    "method": "text",
    "sourceId": 1
}
```

> **注意**：`sourceId` 支持数字类型（对应消息源配置中的 SourceId），也可以使用字符串类型匹配 Name 字段。

**参数说明**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `terminalId` | string | 是 | 终端标识，用于匹配推送配置 |
| `messageType` | string | 是 | 消息类型，固定为 `push` |
| `method` | string | 是 | 推送方式：`text`（文字）、`image`（图片） |
| `sourceId` | number/string | 否 | 消息源ID，指定后自动获取数据推送（推荐使用数字） |
| `sourceName` | string | 否 | 消息源名称（兼容旧版本，中文可能有编码问题） |

**响应**

```json
{
    "success": true,
    "data": {
        "id": 1,
        "terminalId": "terminal_001",
        "messageType": "push",
        "method": "image",
        "status": "Pending",
        "receivedAt": "2024-01-01T10:00:00",
        "createdAt": "2024-01-01T10:00:00"
    },
    "message": "消息请求已记录"
}
```

## 四、数据库表结构

### 4.1 FeishuMessages 表

```sql
CREATE TABLE IF NOT EXISTS FeishuMessages (
    Id INTEGER PRIMARY KEY AUTOINCREMENT,
    MessageType TEXT NOT NULL,
    TerminalId TEXT NOT NULL,
    Method TEXT NOT NULL,
    Status TEXT NOT NULL DEFAULT 'Pending',
    ReceivedAt TEXT NOT NULL,
    ProcessedAt TEXT,
    CreatedAt TEXT NOT NULL
);

-- 索引
CREATE INDEX idx_feishu_terminal ON FeishuMessages(TerminalId);
CREATE INDEX idx_feishu_status ON FeishuMessages(Status);
CREATE INDEX idx_feishu_received ON FeishuMessages(ReceivedAt);
```

### 4.2 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `Id` | INTEGER | 主键，自增 |
| `MessageType` | TEXT | 消息类型：`push` |
| `TerminalId` | TEXT | 终端标识 |
| `Method` | TEXT | 推送方式：`text`、`image` |
| `Status` | TEXT | 状态：`Pending`、`Sent`、`Failed` |
| `ReceivedAt` | TEXT | 接收时间（ISO 8601） |
| `ProcessedAt` | TEXT | 处理时间（ISO 8601） |
| `CreatedAt` | TEXT | 创建时间（ISO 8601） |

### 4.3 状态流转

## 五、后台扫描服务

### 5.1 工作原理

`FeishuPushService` 使用 `Timer` 定时器定期扫描数据库中的待处理消息。

```csharp
public void Start()
{
    var config = _configService.Config.FeishuPushConfig;
    var intervalMs = config.ScanIntervalSeconds * 1000;

    _scanTimer = new Timer(ScanCallback, null, 0, intervalMs);
}
```

### 5.2 扫描逻辑

```csharp
private void ScanAndProcess()
{
    // 1. 获取待处理消息（最多50条）
    var pendingMessages = service.GetMessagesByStatus("Pending", 50);

    foreach (var message in pendingMessages)
    {
        // 2. 根据 TerminalId 查找对应的推送配置
        var terminalConfig = FindConfig(message.TerminalId);

        if (terminalConfig == null)
        {
            // 未找到配置，标记失败
            service.UpdateMessageStatus(message.Id, "Failed", DateTime.Now);
            continue;
        }

        // 3. 根据配置发送消息
        var success = true;
        var errorMsg = "";

        // 发送文字消息
        if (message.Method == "text" && !string.IsNullOrWhiteSpace(terminalConfig.TextWebhook))
        {
            success = SendTextMessage(terminalConfig.TextWebhook, "hello");
        }

        // 发送图片消息
        if (message.Method == "image")
        {
            success = SendImageMessage(...);
        }

        // 4. 更新状态
        var newStatus = success ? "Sent" : "Failed";
        service.UpdateMessageStatus(message.Id, newStatus, DateTime.Now);
    }
}
```

## 八、图片消息推送

### 8.1 推送流程

```
步骤1: 获取 AccessToken (OAuth 认证)
    ↓
步骤2: 上传图片获取 image_key
    ↓
步骤3: 发送图片消息
```

### 8.2 前置配置（飞书开放平台）

| 步骤 | 操作 |
|------|------|
| 1 | 访问 [飞书开放平台](https://open.feishu.cn/app) 创建企业自建应用 |
| 2 | 获取 `App ID` 和 `App Secret` |
| 3 | 添加「机器人」应用能力 |
| 4 | 权限管理 → 添加 `im:message` 权限 |
| 5 | 版本管理与发布 → 创建版本 → 发布 |
| 6 | 将机器人添加到目标群聊 |

### 8.3 获取接收者 ID

| 类型 | 获取方式 | ID 格式 |
|------|----------|----------|
| 群组 | 群设置 → 群信息 → 复制群 ID | `oc_` 开头 |
| 用户 | 用户资料 → 复制用户 ID | `ou_` 开头 |

### 8.4 核心代码

#### 获取 AccessToken

```csharp
private string? GetTenantAccessToken(string apiKey, string secretKey)
{
    var payload = new { app_id = apiKey, app_secret = secretKey };
    var json = JsonConvert.SerializeObject(payload);
    var content = new StringContent(json, Encoding.UTF8, "application/json");

    var response = _httpClient.PostAsync(
        "https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal",
        content
    ).Result;

    var result = JsonConvert.DeserializeObject<TokenResponse>(responseBody);
    return result?.TenantAccessToken;
}
```

#### 上传图片

```csharp
private string? UploadImage(string token, string imagePath)
{
    var imageBytes = File.ReadAllBytes(imagePath);

    using var content = new MultipartFormDataContent();
    var imageContent = new ByteArrayContent(imageBytes);
    imageContent.Headers.ContentType = new MediaTypeHeaderValue("image/png");
    content.Add(imageContent, "image", Path.GetFileName(imagePath));
    content.Add(new StringContent("message"), "image_type");  // 必须是 "message"

    var request = new HttpRequestMessage(HttpMethod.Post,
        "https://open.feishu.cn/open-apis/im/v1/images");
    request.Headers.Authorization = new AuthenticationHeaderValue("Bearer", token);
    request.Content = content;

    var response = _httpClient.SendAsync(request).Result;
    var result = JsonConvert.DeserializeObject<UploadResponse>(responseBody);

    return result?.Data?.ImageKey;
}
```

#### 发送图片消息

```csharp
private bool SendImageByKey(string token, string imageKey, string receiverId)
{
    var payload = new
    {
        receive_id = receiverId,
        msg_type = "image",
        content = JsonConvert.SerializeObject(new { image_key = imageKey })
    };

    // ID 类型判断
    var idType = receiverId.StartsWith("ou_") ? "open_id" : "chat_id";
    var url = $"https://open.feishu.cn/open-apis/im/v1/messages?receive_id_type={idType}";

    var request = new HttpRequestMessage(HttpMethod.Post, url);
    request.Headers.Authorization = new AuthenticationHeaderValue("Bearer", token);
    request.Content = new StringContent(JsonConvert.SerializeObject(payload),
        Encoding.UTF8, "application/json");

    var response = _httpClient.SendAsync(request).Result;
    return JsonConvert.DeserializeObject<FeishuResponse>(responseBody)?.Code == 0;
}
```

## 十、常见错误排查

| 错误码 | 错误信息 | 原因 | 解决方案 |
|--------|----------|------|----------|
| - | `消息请求已记录` 但未收到推送 | 终端配置缺失 | 检查 `terminalId` 是否匹配 |
| 99991672 | receive_id_type is required | URL 缺少参数 | 添加 `?receive_id_type=chat_id` |
| 99992351 | Invalid open_id or chat_id | ID 类型判断错误 | `oc_` 用 chat_id，`ou_` 用 open_id |
| 230002 | 参数错误 | JSON 格式错误 | content 需是 JSON 字符串 |
| 230029 | 权限不足 | 未添加 im:message 权限 | 在开放平台添加权限 |
| 230038 | 应用未发布 | 应用未发布 | 发布应用新版本 |
| 230044 | 机器人能力未启用 | 未添加机器人 | 添加「机器人」能力 |
| 234001 | Invalid request param | image_type 格式错误 | 作为表单字段而非 URL 参数 |

## 十四、消息源配置

### 14.1 功能说明

消息源配置允许您预先配置多个 API 数据源，系统会根据请求中的 `sourceId` 自动调用对应的 API，获取数据后格式化并推送到飞书。

### 14.2 配置步骤

1. 在主界面点击「消息源设置」
2. 点击「添加」创建新消息源
3. 填写配置信息：
   - **SourceId**：唯一标识符（推荐使用简单字符串或数字）
   - **名称**：显示名称
   - **API 地址**：数据接口 URL
   - **请求方式**：GET 或 POST
   - **响应格式**：可选，用于格式化输出

### 14.3 示例配置

假设配置如下消息源：

| SourceId | 名称 | API 地址 |
|----------|------|----------|
| `1` | 获取多空排名 | `https://api.example.com/ranking` |

### 14.4 使用示例

```bash
# 调用 sourceId=1 的消息源
curl -X POST http://localhost:5000/api/feishu/message \
  -H "Content-Type: application/json; charset=utf-8" \
  -d '{
    "terminalId": "t1",
    "messageType": "push",
    "method": "text",
    "sourceId": 1
  }'
```

系统会自动：
1. 根据 `sourceId=1` 查找消息源配置
2. 调用配置的 API 地址获取数据
3. 按响应格式格式化数据
4. 推送到飞书

> **提示**：使用数字 `sourceId` 可以避免中文编码问题，推荐优先使用数字 ID。
